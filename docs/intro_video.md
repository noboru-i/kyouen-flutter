# 詰め共円アプリ紹介動画 — HyperFrames 制作指示書

このドキュメントは `kyouen-app-intro/index.html` の現在の実装を基準にした制作仕様です。
今後はこのファイルを先に更新し、その内容を Coding Agent で `index.html` に反映してください。

## セットアップ

```bash
cd kyouen-app-intro

# プレビュー
npm run dev

# 検証
npm run check

# レンダリング
npm run render
```

`npm run render` には FFmpeg が必要です。未インストールの場合は `brew install ffmpeg` を実行してください。

## 動画仕様

| 項目 | 値 |
|---|---|
| 総尺 | 60秒 |
| 解像度 | 3840x2160 |
| フレームレート | 30fps |
| フォーマット | MP4 |
| 対象ストア | Google Play |
| BGM | `kyouen-app-intro/assets/intro-bgm.mp3` |

## BGM

生成したBGMは、HyperFramesの素材ディレクトリである以下に配置する。

```text
kyouen-app-intro/assets/intro-bgm.mp3
```

`hyperframes.json` で `paths.assets` が `assets` に設定されているため、動画用の音声・画像・動画素材は `kyouen-app-intro/assets/` 配下で管理する。

### 音声ファイル仕様

- 推奨ファイル名: `intro-bgm.mp3`
- 推奨形式: MP3 または WAV
- 推奨尺: 60秒
- ボーカル・歌詞なし
- 効果音が目立ちすぎないBGMにする
- CTAの余韻を残すため、最後は急に切らず自然にフェードアウトする
- ナレーションを追加する可能性を考慮し、音量は控えめにする

### HTMLに組み込む場合

`kyouen-app-intro/index.html` からは次のように参照する。

```html
<audio
  src="assets/intro-bgm.mp3"
  data-start="0"
  data-duration="60"
  data-track-index="0"
></audio>
```

BGMを差し替える場合も、原則として同じファイル名 `intro-bgm.mp3` で上書きする。別名で管理する場合は、このドキュメントと `index.html` の参照パスを合わせて更新する。

## デザイン方針

- トーン: 落ち着いた、静的、ミニマル
- 背景: 単色 `#0D1117`
  - MP4圧縮時のバンディングを避けるため、背景グラデーションは使わない
- メインテキスト: `#e8e8e8`
- サブテキスト: `#888888`
- アクセント: `#FF6B35`
- 盤面:
  - 背景: `#4CAF50` から `#2E7D32` の盤面内グラデーション
  - 罫線: 暗い緑/黒系の細線
- 石:
  - 黒石: `#4A4A4A -> #1C1C1C -> #000000` の radial gradient
  - 白石: `#FFFFFF -> #E8E8E8 -> #D0D0D0` の radial gradient
  - 石は重ならないサイズにする
- 共円円:
  - 色: `#FF6B35`
  - strokeWidth: `4`
  - strokeLinecap: `round`
  - 0時方向から時計回りに描画する

## 実装メモ

- `kyouen-app-intro/index.html` のルートは4K出力用に以下を維持する。

```html
<div id="root" data-composition-id="main" data-start="0" data-duration="60" data-width="3840" data-height="2160">
```

- 既存のシーンレイアウトは1280x720の座標系で管理し、`#stage` を3倍スケールして4Kキャンバスに配置する。

```css
#root {
  width: 3840px;
  height: 2160px;
}

#stage {
  width: 1280px;
  height: 720px;
  transform: scale(3);
  transform-origin: top left;
}
```

- 背景はCSS変数で単色管理する。

```css
:root {
  --bg: #0D1117;
}

html,
body,
#s1,
#s2a,
#s2b,
#s2c,
#s3a,
#s3b,
#s3c,
#s4 {
  background: var(--bg);
}
```

- アニメーションはGSAP timelineで管理する。
- シーン切り替えは blur + scale + opacity のトランジションを使う。
- 検証は必ず `npm run check` で行う。

## シーン構成

### Scene 1 — 導入

**時間**: 0s 〜 約4s

**目的**: タイトルを短く提示する。

**テキスト**:

```text
詰め共円
― 共円パズル ―
```

**レイアウト**:

- 画面中央にタイトル
- サブタイトルをタイトル下に配置
- 幾何学モチーフとして薄い装飾円は使ってよい

**注意**:

- 内容量が少ないため長くしすぎない
- 余った時間はScene 4に回す

### Scene 2A — 盤面説明

**時間**: 約4s 〜 約11s

**目的**: ゲーム開始時の状態を説明する。

**テキスト**:

```text
盤面に、石が並んでいます。
```

サブテキスト:

```text
オセロに似た碁盤目の上に、黒と白の石が置かれた状態からゲームは始まります。
```

### Scene 2B — 共円の定義

**時間**: 約11s 〜 約20s

**目的**: 「共円」の意味を説明する。

**テキスト**:

```text
「共円」とは ―
```

説明:

```text
同一円周上に存在する 4つの点のことです。
```

補足:

```text
3つの点なら、どんな並びでも円として見ることができます。
4点目も同じ円の上にあれば、共円の関係になります。
```

**右側の図**:

- 4つの石をすべて選択状態の白石として表示する
- 背景の薄い円は表示しない
- オレンジの共円円を描画する
- 共円円は0時方向から時計回りに描画する

### Scene 2C — ゲームの目標

**時間**: 約20s 〜 約27s

**目的**: プレイヤーが何を探すのかを説明する。

**テキスト**:

```text
GAME GOAL
```

```text
盤面の中から、共円になる 4つの石を見つけてください。
```

補足:

```text
正方形・長方形・等脚台形・直線など、幾何学的なパターンが手がかりになります。
```

**注意**:

- このアプリでは直線も円の一種として扱う。

### Scene 3A — 問題を開く

**時間**: 約27s 〜 約37s

**目的**: 問題を選んで盤面を開く流れを見せる。

**左テキスト**:

```text
STEP 1
```

```text
問題を選んで盤面を開きます。
```

サブテキスト:

```text
いろいろな盤面から、気になる問題を選べます。
```

**右モックアップ**:

指定の4ステージを順番に切り替える。

```js
const stageStrings = [
  "000000010000001100001100000000001000",
  "000000000000000100010010001100000000",
  "000000001000010000000100010010001000",
  "001000001000000010010000010100000000",
];
```

**盤面仕様**:

- 6x6 のステージ文字列として扱う
- `0`: 空
- `1`: 石
- SVGで盤面を描画する
- 石のサイズは重ならないようにする
- 罫線と石の座標系は、実アプリ画面に見える配置に合わせる
- このシーンでは選択状態へ変更しない
- このシーンでは共円円を描かない

### Scene 3B — 石を選択する

**時間**: 約37s 〜 約46s

**目的**: 4つの石を選択する操作を見せる。

**左テキスト**:

```text
STEP 2
```

```text
4つの石をタップして選択します。
```

サブテキスト:

```text
選んだ黒石は白く変わります。
```

**右モックアップ**:

- Scene 3Aのステージ4を表示する
- 選択対象は次の4点

```js
const selectedStage4 = [8, 16, 19, 27];
```

- 選択演出はオレンジ外周円ではなく、黒石が白石に変わる表現にする
- 4つの石を順番に白くする

### Scene 3C — 正解判定

**時間**: 約46s 〜 約54s

**目的**: 判定ボタンを押して、共円が成立することを見せる。

**左テキスト**:

```text
STEP 3
```

```text
判定ボタンを押すと、正解かどうかをチェックします。
```

サブテキスト:

```text
共円なら、円が描かれてクリアです。
```

**右モックアップ**:

- Scene 3Bと同じステージ4を表示する
- `selectedStage4 = [8, 16, 19, 27]` の4石は白石で表示する
- ボタン文言は以下にする

```text
共円！！
```

- ボタン押下後、オレンジの共円円を描画する
- 共円円は0時方向から時計回りに描画する

### Scene 4 — 締め・CTA

**時間**: 約54s 〜 60s

**目的**: アプリ名とGoogle Play配信を提示する。

**テキスト**:

```text
詰め共円は、幾何学的な直感を静かに問いかけるパズルです。
```

```text
詰め共円
```

```text
Google Play にて配信中
```

**レイアウト**:

- 中央寄せ
- キャッチコピーは1行で表示する
- アプリアイコンやプレースホルダーアイコンは表示しない
- Scene 1を短くした分、Scene 4はやや長めに見せる

## 盤面描画の仕様

### ステージ文字列

ステージ文字列は6x6として読み取る。

```js
const row = Math.floor(index / 6);
const col = index % 6;
```

### 石

- 石は盤面内で重ならないサイズにする
- 黒石/白石はradial gradientで立体感を出す
- 選択状態は白石で表現する

### 罫線

- 罫線は実アプリの見た目に合わせる
- 石の位置と罫線の位置がずれないよう、同じ座標系で管理する
- 盤面の正しさを確認する時は、ステージ4の見た目を優先する

### ステージ4の選択点

Scene 3B / 3C の選択対象は以下。

```js
const selectedStage4 = [8, 16, 19, 27];
```

この4点で共円円を描く。

## BGM生成プロンプト

以下のプロンプトにより、GoogleのLyriaにより生成。

```text
60秒のミニマルなパズルアプリ紹介動画向けのBGM。

雰囲気は、静かで知的、落ち着いた、幾何学的、少し神秘的。
暗い単色背景に、碁盤目、黒石と白石、オレンジ色の円が描かれていく映像に合う音楽。

テンポは中低速、約80〜95 BPM。
派手すぎず、広告っぽくなりすぎず、集中感と上品さがある。
数学パズル、ロジック、思考、静かな達成感を感じさせる。

構成:
0〜4秒: 余白のある静かな導入。タイトル表示に合わせて控えめに始まる。
4〜27秒: 盤面と共円の説明に合わせて、柔らかいパルスや繰り返しのミニマルなフレーズ。
27〜46秒: 問題選択と石の選択シーン。少しだけリズム感を増やし、操作感を出す。
46〜54秒: 正解判定でオレンジの円が描かれる場面。控えめな盛り上がりと透明感のある解決感。
54〜60秒: アプリ名とGoogle Play配信中のCTA。静かに余韻を残して終わる。

楽器:
ソフトなピアノ、マリンバまたはグロッケンの軽い音、控えめなシンセパッド、細い電子パルス、深すぎないアンビエントベース。
ドラムは最小限。強いキックや派手なスネアは使わない。

避けたい要素:
激しいEDM、ロック、オーケストラの大仰な盛り上がり、コミカルな音、和風すぎる音色、ボーカル、歌詞、効果音が目立つ構成。

ループ感はあってよいが、60秒の映像として自然な始まりと終わりを持たせる。
全体として、静かな思考と「共円を発見する気持ちよさ」が伝わるBGM。
```
